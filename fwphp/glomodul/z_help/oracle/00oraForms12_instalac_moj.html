    1. Node Manager is needed to "manage" the node. Meaning, if you expect to use Fusion Middleware Control (FMC) to make configuration changes and control servers you will need it running. That said, it is not supported to edit most configurations outside of FMC. So unlike old-school where you used your favorite text editor and made changes to formsweb.cfg (as an example), you cannot/should not do that in 12c. All changes that can be made in FMC should be made from within FMC !!.

    2. Unless you plan to configure the managed server(s) to start automatically with the machine or as a Windows Service, it likely would be better to start the managed servers from with FMC. This will also avoid having extra DOS shells open.

    3. DOMAIN_HOME\config\fmwconfig\tnsnames.ora file is not managed by FMC and therefore can be edited in a text editor. This file is used by the Forms and Reports Builders, compilers, etc, and runtime components. So it is not recommended that you move this file. However, if you decide to move the file you will need to update the following (at least for Forms):
    Setting TNS_ADMIN in the Registry will only work for the client tools (Builder, compiler, etc). It will not be used by runtime.
    HKEY_LOCAL_MACHINE\SOFTWARE\ORACLE\KEY_OracleHome<SOME NUMBER>
    The Forms Runtime includes a TNS_ADMIN reference in its Environment Variable configuration (aka default.env). To update the value, use FMC

      Database running from a different server, that’s how you reduce the clutter. Just make sure that you have both server’s host files having entries for each other, do ping to insure that both machines respond to each other.
      Once you confirmed the connectivity, all you need to do is to either copy the tnsnames.ora file from the database server to Weblogic’s tns location or editing the existing tnsnames.ora and adding the entries manually.
