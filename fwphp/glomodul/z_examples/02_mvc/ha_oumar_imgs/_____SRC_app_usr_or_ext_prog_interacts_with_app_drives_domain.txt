# DDD HA - Hexagonal architecture also called PORTS & ADAPTERS
HA is a case (event, occurrence) of application of DDD (Domain Driven Design).  
HA permits to SEPARATE app code (module code as Oracle Forms .fmb) in three parts :  
1. App = application (module) code, **drives domain**. User or the external programs interact with app.
2. D - domain = business logic code, **drives infrastructure**
3. I - infrastructure code - dependencies that app needs to work.

## Code flow :
1. src/Controller/GalleryController.php
2. src/Domain/GalleryManager.php
3. src/Infrastructure/GalleryRepository.php
4. src/Infrastructure/GalleryDriver.php

### APPLICATION CODE
APPLICATION IS CONTROLLER in dirs Config and Controllers  (**Home_ctr**)  

App ctr is left side code by witch usr or external programs interact with web site app. In dirs Config and Controllers are :  
   - code of graphical interface (assets in zinc dir, templates = view scripts in module dir),   
   - routes (array in **Home_ctr**)  
   - controllers (methods in Home_ctr), request, response, config (**Config_allsites**)...  
   - everything that is not part of D and I  
App is isolated from D and I and must be able to be STARTED BY USER, PROGRAM, OR CLI (BASH) SCRIPT.

2. automatically test  core business behavior independently of everything else
